# Инструкция: Массовая проверка всех идей

## Цель
Прочитать все папки в репозитории `my-ideas` и проверить каждую идею на живучесть по единой схеме.

## Шаги

### Шаг 1. Собрать список идей
- Пройди по всем папкам в `my-ideas`.
- Игнорируй системные папки: `agents/`, `research/`, `business/`, `posts/`.
- Для каждой папки считай, что это отдельная идея/проект.
- Если в папке есть `MASTER.md` — используй его как описание идеи.
- Если нет — используй название папки как название идеи.

### Шаг 2. Проверить каждую идею по 5 критериям
Для каждой идеи оцени:

1. **Спрос** — есть ли аналоги? Кто ищет это?
2. **Ресурсы** — что уже есть (код, токены, доступы)?
3. **Сложность** — от 1 до 10 (1 — просто, 10 — очень сложно).
4. **Риски** — что может пойти не так?
5. **Польза** — что я получу, если реализую?

### Шаг 3. Сделать общий отчёт
Создай файл `проверка-всех-идей.md` в корне `my-ideas` со структурой:

```
# Результаты проверки всех идей

## 📊 Сводная таблица
| Идея | Спрос | Ресурсы | Сложность | Риски | Польза | Вердикт |
|------|-------|---------|-----------|-------|--------|---------|
| ...  | ...   | ...     | ...       | ...   | ...    | ...     |

## 🔍 Детальный разбор каждой идеи
### [Название папки/идеи]
- **Описание:** ...
- **Спрос:** ...
- **Ресурсы:** ...
- **Сложность:** ...
- **Риски:** ...
- **Польза:** ...
- **Вердикт:** зелёная / жёлтая / красная

### [следующая идея]...

## 🏆 Топ-3 идеи для реализации
1. [название] — почему
2. [название] — почему
3. [название] — почему
```

### Шаг 4. Отправить отчёт
- Сохрани файл `проверка-всех-идей.md` в корне `my-ideas`.
- Скажи мне: «Готово. Отчёт в проверка-всех-идей.md».
